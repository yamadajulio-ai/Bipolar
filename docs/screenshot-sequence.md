# Screenshot Sequence + Demo Video Script

App Store submission assets for Suporte Bipolar.

## Screenshot Sequence (6.7" and 6.5" required)

Best sequence based on auditor consensus (8 screenshots):

1. **Dashboard /hoje** — Mood + sleep + stability score. Shows the full daily overview with the stability score widget, risk radar, and "Seu estado hoje" section. First impression of the app's value.
2. **Check-in** — Mood slider with haptic feedback UI. Demonstrates the core daily interaction: selecting mood, energy, and sleep data.
3. **Insights + AI Narrative** — Graphs + weekly AI-generated summary. Shows Recharts visualizations and the personalized narrative powered by GPT-5.4.
4. **Sleep History** — Color-coded cards with duration bars. Displays the sleep tracking system with faixas de cor (green/amber/red/purple) and multi-cycle support.
5. **SOS Mode** — Emergency contacts + grounding exercise. Demonstrates crisis support: SAMU 192, CVV 188, personal emergency contacts, and guided grounding.
6. **Weekly Assessment** — PHQ-9/ASRM scales. Shows the clinical self-assessment wizard with validated psychiatric instruments.
7. **Biometria** — Face ID lock screen. Demonstrates the native security layer protecting sensitive health data.
8. **Account** — Export/delete + theme toggle. Shows LGPD compliance (data export, account deletion) and dark mode support.

### Screenshot Guidelines

- Capture on iPhone 15 Pro Max (6.7") and iPhone 14 Plus (6.5") simulators.
- Use real sample data (demo account), not empty states.
- Dark mode screenshots are optional but recommended as alternates.
- No status bar overlays — App Store Connect crops automatically.
- Portuguese (pt-BR) captions above each screenshot if using promotional text frames.

## Demo Video Script (1 minute)

For App Store Review Notes — demonstrates native functionality and real user flows.

### Video Production Notes

- Record on physical iPhone (not simulator) for authentic haptic/Face ID experience.
- 1080x1920 portrait orientation, 30fps minimum.
- No voiceover required — use on-screen captions in pt-BR.
- Keep transitions smooth, no cuts longer than 0.5s.
- Ensure demo account has 7+ days of realistic sample data.
- Upload as App Preview in App Store Connect (up to 30s per preview, or single 1-min video in Review Notes attachment).
